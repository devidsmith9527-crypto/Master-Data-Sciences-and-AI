១. ការពិពណ៌នាមុខវិជ្ជា (Course Description)
មុខវិជ្ជានេះត្រូវបានរចនាឡើងយ៉ាងពិសេសសម្រាប់និស្សិតថ្នាក់អនុបណ្ឌិត ដើម្បីផ្តល់ចំណេះដឹងស៊ីជម្រៅលើការរចនា និងវិភាគក្បួនដោះស្រាយ (Algorithm Design) សម្រាប់ Artificial Intelligence (AI) & Data Science ក្នុងបរិបទប្រទេសកម្ពុជា។ ចំណុចផ្តោតសំខាន់រួមមាន ការបង្កើនប្រសិទ្ធភាពកូដ (Code Efficiency), កាត់បន្ថយការប្រើប្រាស់អង្គចងចាំ (Memory Management), និងការដោះស្រាយបញ្ហាស្មុគស្មាញ (Time and Space Complexity) តាមរយៈភាសា Python។
សទ្ទានុក្រម និងពាក្យគន្លឹះ (Key Terms)

Data Structures (រចនាសម្ព័ន្ធទិន្នន័យ)

វិធីសាស្ត្រក្នុងការរៀបចំ និងផ្ទុកទិន្នន័យនៅក្នុងអង្គចងចាំ។ ឧទាហរណ៍៖ Array, Stack, Queue, Hash Table, Trees។

Time / Space Complexity

Time Complexity: រយៈពេលដំណើរការ។ Space Complexity: ទំហំ Memory ដែលកូដប្រើប្រាស់ប្រៀបធៀបនឹង Big-O Notation។

Algorithms (ក្បួនដោះស្រាយ)

សំណុំនៃជំហានដោះស្រាយបញ្ហាច្បាស់លាស់។ ឧទាហរណ៍៖ ការតម្រៀប (Sorting) និងការស្វែងរក (Searching)។

AI / ML / NLP / OOP

AI: Artificial Intelligence, ML: Machine Learning, NLP: Natural Language Processing, OOP: Object-Oriented Programming។

២. លទ្ធផលសិក្សារំពឹងទុក (Course Learning Outcomes - CLO)

ក្រោយបញ្ចប់ការសិក្សាលើមុខវិជ្ជានេះ និស្សិតថ្នាក់អនុបណ្ឌិតនឹងទទួលបានសមត្ថភាព និងលទ្ធផលរំពឹងទុកដូចខាងក្រោម៖

លទ្ធផលសិក្សានៃមុខវិជ្ជា (CLO) លទ្ធផលនៃកម្មវិធីអប់រំ​ (PLO) ដែលត្រូវបានវាស់វែង
CLO 1
(ការវិភាគកូដ)		 វិភាគ និងវាយតម្លៃភាពស្មុគស្មាញនៃកូដ (Time and Space Complexity, Big-O Notation) ឱ្យស្របតាមស្តង់ដារការសរសេរកូដ Object-Oriented Programming (OOP) នៅក្នុងភាសា Python។
CLO 2
(រចនាសម្ព័ន្ធលីនេអ៊ែរ)		 អនុវត្តការរៀបចំទិន្នន័យដោយប្រើប្រាស់ Linear Data Structures ដូចជា Arrays, Linked Lists, Stacks (LIFO), និង Queues (FIFO) ដើម្បីគ្រប់គ្រងអង្គចងចាំប្រកបដោយប្រសិទ្ធភាព។
CLO 3
(Non-Linear)		 ដោះស្រាយបញ្ហាបណ្តាញទិន្នន័យធំៗដែលមានភាពស្មុគស្មាញដោយប្រើប្រាស់ Trees (Binary Search Tree - BST), Hash Tables កម្រិតខ្ពស់, និង Graphs (BFS, DFS)។
CLO 4
(Optimization & Sorting)		 អភិវឌ្ឍប្រព័ន្ធ Artificial Intelligence (AI) និងការតម្រៀប (Sorting & Searching Algorithms ដូចជា Quick Sort, Merge Sort) ប្រកបដោយប្រសិទ្ធភាពល្បឿនខ្ពស់។
CLO 5
(Dynamic Programming)		 ប្រើប្រាស់បច្ចេកទេស Dynamic Programming (DP) និង Greedy Algorithms ដើម្បីស្វែងរកផ្លូវខ្លីបំផុត ដោះស្រាយបញ្ហាអាជីវកម្ម និងស្រាវជ្រាវ (Machine Learning / NLP)។

    1. Logistics Optimization (CLO 3,4): អនុវត្ត Dijkstra's Algorithm ដើម្បីស្វែងរកផ្លូវដឹកជញ្ជូនដែលខ្លី និងលឿនបំផុត (Delivery Route) នៅក្នុងក្រុងសៀមរាប ឬរាជធានីភ្នំពេញ។
    2. Customer Data Retrieval (CLO 3,5): រចនាប្រព័ន្ធផ្ទុកទិន្នន័យអតិថិជនរាប់សែននាក់ សម្រាប់វិស័យធនាគារក្នុងស្រុក ដោយប្រើ Hash Table ផ្ទាល់ខ្លួន ដើម្បីទាញយកទិន្នន័យក្នុងល្បឿន O(1)។
    3. Khmer NLP Processing (CLO 3,5): ប្រើប្រាស់ Tries Data Structure ដើម្បីកសាងប្រព័ន្ធកាត់ពាក្យភាសាខ្មែរ (Word Tokenizer) សាមញ្ញ ឬប្រព័ន្ធទស្សន៍ទាយពាក្យ (Auto-complete) ដោយប្រើ Artificial Intelligence (AI)។
    4. Tourism Recommendation (CLO 3,5): អនុវត្ត K-D Trees (K-Dimensional Trees) ដើម្បីបង្កើត Recommendation System សម្រាប់ណែនាំសណ្ឋាគារ ឬកន្លែងកម្សាន្តនៅតំបន់អង្គរ ដោយផ្អែកលើទីតាំង GPS។
    5. Resource Allocation (CLO 4): ដោះស្រាយបញ្ហា Knapsack Problem ដោយប្រើ Dynamic Programming (DP) ដើម្បីរៀបចំគម្រោងថវិកា ឬការបែងចែកធនធានកសិកម្មប្រកបដោយប្រសិទ្ធភាពចំណាយខ្ពស់បំផុត។
  • – ភារកិច្ចនិស្សិត៖ ធ្វើការជាក្រុម (២-៣ នាក់), សរសេរកូដ Data Structures and Algorithms (DSA) ជាភាសា Python និងឡើងធ្វើបទបង្ហាញគម្រោងទាំងមូលក្នុងថ្នាក់។

៥.២.២- កិច្ចការពិភាក្សាជាក្រុមតូចៗនៅក្នុងថ្នាក់ (Group discussion)

  • · កាលបរិច្ឆេទ ៖ អនុវត្តជាប្រចាំ។ វាយតម្លៃបូកបញ្ចូលក្នុងពិន្ទុវត្តមាន ១០%។
  • · ប្រធានបទ ៖ ពិភាក្សាពីភាពខុសគ្នារវាង Array vs Linked List, វិធីដោះស្រាយ Hash Table Collisions, និងការប្រៀបធៀប BFS vs DFS Search លើបណ្តាញទំនាក់ទំនងសង្គម។

៥.៣- ប្រឡងពាក់កណ្តាលឆមាស (Midterm Exam)

  • · កាលបរិច្ឆេទ ៖ ប្រឡងនៅថ្ងៃអាទិត្យ នៃ សប្តាហ៍ទី ៣ (មានពិន្ទុ ២០%)
  • · ប្រធានបទ ៖ ប្រឡងទ្រឹស្តី និងកូដ (គ្របដណ្តប់ពីជំពូក១ ដល់ទី ៥) រយៈពេល ៩០នាទី ទៅ ១៨០នាទី ។

៥.៤- ប្រឡងបញ្ចប់ឆមាស (Final Exam)

  • · កាលបរិច្ឆេទ ៖ ប្រឡងនៅថ្ងៃអាទិត្យ នៃ សប្តាហ៍ទី ៦ (មានពិន្ទុ ៤០%)
  • · ប្រធានបទ ៖ គ្របដណ្តប់មេរៀនទាំងមូល (ជំពូក១ ដល់ទី ៨) សម្រាប់ការវាយតម្លៃសមត្ថភាពជារួម។ រយៈពេល ១២០ នាទី។
