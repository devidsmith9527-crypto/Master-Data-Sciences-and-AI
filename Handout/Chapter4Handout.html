១. សេចក្តីផ្តើមអំពី Linked Lists (Scattered Memory)
Linked List គឺជាប្រភេទរចនាសម្ព័ន្ធទិន្នន័យជាជួរ (Linear Data Structure) ដ៏មានឥទ្ធិពល ដែលធាតុនីមួយៗ (ហៅថា Node) មិនតម្រូវឲ្យរក្សាទុកនៅទីតាំងអង្គចងចាំជាប់ៗគ្នា (Contiguous Memory) ដូចជា Array នោះឡើយ។ Nodes អាចស្ថិតនៅទីកន្លែងរាយប៉ាយគ្រប់ទិសទីនៅក្នុង Memory (RAM) ដោយពួកវាប្រើប្រាស់ Pointer / Reference (ខ្សែចង្អុលអាសយដ្ឋាន) ដើម្បីចងភ្ជាប់គ្នាពីមួយទៅមួយជារលកបន្តបន្ទាប់។
ការដោះស្រាយបញ្ហា Memory Allocation និង Fragmentation
ដែនកំណត់របស់ Arrays
Array ត្រូវការប្លុក Memory ទំហំធំ និងជាប់គ្នាជានិច្ច។ ប្រសិនបើ RAM របស់កុំព្យូទ័រមានចន្លោះទំនេរតូចៗនៅរាយប៉ាយ (Memory Fragmentation) សរុបទៅមានទំហំធំល្មម ប៉ុន្តែគ្មានប្លុកណាមួយជាប់គ្នាធំល្មមសម្រាប់ Array នោះទេ កុំព្យូទ័រនឹងបរាជ័យក្នុងការបង្កើត Array (Out of Memory Error)។
ដំណោះស្រាយរបស់ Linked Lists
Linked List មិនខ្វល់ខ្វាយពីការ Fragment នៃ Memory ឡើយ។ ឲ្យតែមានចន្លោះទំនេរតូចមួយនៅក្នុង RAM វានឹងបង្កើត Node ថ្មីនៅទីនោះ ហើយចង Pointer ពី Node មុនមកកាន់ទីតាំងថ្មីនេះ។ នេះជាយន្តការបែងចែក Memory បត់បែន (Dynamic Memory Allocation) ដ៏ប្រសើរ។
ដែនកំណត់ធំរបស់ Linked Lists៖ គ្មាន Random Access ឡើយ

ដោយសារតែទិន្នន័យមិនស្ថិតនៅលើទីតាំងជាប់គ្នា យើងមិនអាចប្រើរូបមន្តគណិតវិទ្យាដើម្បីលោតទៅយកទិន្នន័យតាមរយៈ Index ដូចជា arr[5] បានលឿនកម្រិត O(1) នោះទេ។ ប្រសិនបើចង់បានទិន្នន័យទី ៥ ក្នុង Linked List យើងគ្មានជម្រើសអ្វីក្រៅពីចាប់ផ្តើមដើរពី Node ដំបូងគេ (Head) ហើយចម្លងតាម Pointer ម្តងមួយជំហានៗរហូតដល់ទី ៥ ឡើយ ដែលប្រើពេលយឺតកម្រិត O(n)។

ករណីសិក្សាទី១៖ ខ្សែរថភ្លើង (Train Analogy)

ទូរថភ្លើងនីមួយៗតំណាងឲ្យ Node ដែលមានផ្ទុកអ្នកដំណើរ (Data) និងទំពក់ដែកនៅចុងទូដើម្បីចងភ្ជាប់ទៅនឹងទូបន្ទាប់ (Next Pointer)។ ក្បាលរថភ្លើងទាំងមូលគឺតំណាងឲ្យ Head Pointer។ ប្រសិនបើទំពក់នៅកណ្តាលរថភ្លើងត្រូវបានផ្តាច់ចេញ ទូទាំងអស់នៅខាងក្រោយនឹងត្រូវដាច់បាត់បង់ចេញពីក្បាលរថភ្លើងភ្លាមៗ។

ករណីសិក្សាទី២៖ ល្បែងដេញតាមតម្រុយ (Treasure Hunt)

អ្នកលេងហ្គេមទទួលបានតែក្រដាសតម្រុយទី១ (Head)។ ក្រដាសនោះប្រាប់ពីទីតាំងកំណប់ទី១ និងមានការណែនាំ (Next) ឲ្យដើរទៅរកក្រដាសតម្រុយទី២។ អ្នកលេងមិនអាចលោតផ្លោះទៅរកកំណប់ចុងក្រោយបានឡើយ គឺត្រូវតែដើរតាមលំដាប់លំដោយរហូតដល់ជួបក្រដាសចុងក្រោយដែលគ្មានការណែនាំបន្តទៀត (None)។

ករណីសិក្សាទី៣៖ កម្មវិធីបញ្ចាំងស្លាយរូបភាពមួយទិស (One-way Slideshow)

នៅក្នុងកម្មវិធីចាក់រូបភាពចាស់ៗ រូបនីមួយៗផ្ទុកតែកូដបង្កប់ចង្អុលទៅយករូបបន្ទាប់។ នៅពេលអ្នកចុច "បន្ទាប់ (Next)" វានឹងរុញទៅមុខជានិច្ច។ ប៉ុន្តែអ្នកមិនអាចចុចប៊ូតុង "ត្រឡប់ក្រោយ (Previous)" បានទេ លុះត្រាតែកម្មវិធីដើរវិលជុំមកដល់រូបភាពទី១ សារជាថ្មី ឬប្រើប្រាស់រចនាសម្ព័ន្ធពីរទិស (Doubly Linked List)។

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None # ដើមឡើយមិនទាន់មានការចង្អុលទៅណាឡើយ

class SinglyLinkedList:
    def __init__(self):
        self.head = None # បង្កើតបញ្ជីទទេមួយដែលមានក្បាលស្មើ None

    # បន្ថែម Node ថ្មីទៅខាងចុងបញ្ជី O(n)
    def insert_at_end(self, data):
        new_node = Node(data)
        if self.head is None:
            self.head = new_node
            return
        # បើមាន Node រួចហើយ ត្រូវដើរទៅរក Tail (Node ចុងក្រោយបង្អស់)
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node

    # បោះពុម្ពទិន្នន័យទាំងអស់ក្នុងបណ្តាញតភ្ជាប់ O(n)
    def print_list(self):
        current = self.head
        elements = []
        while current:
            elements.append(str(current.data))
            current = current.next
        print(" -> ".join(elements) + " -> None")

# ដំណើរការសាកល្បង
llist = SinglyLinkedList()
llist.insert_at_end(10)
llist.insert_at_end(20)
llist.insert_at_end(30)

print("លទ្ធផលការតភ្ជាប់ Singly Linked List:")
llist.print_list()

class DNode:
    def __init__(self, data):
        self.data = data
        self.prev = None # ខ្សែចង្អុលថយក្រោយ
        self.next = None # ខ្សែចង្អុលទៅមុខ

# បង្កើត Node ចំនួន ៣ ដាច់ដោយឡែកពីគ្នា
nodeA = DNode("ទំព័រទី១ (Home)")
nodeB = DNode("ទំព័រទី២ (Products)")
nodeC = DNode("ទំព័រទី៣ (Checkout)")

# ភ្ជាប់ទំនាក់ទំនងពីរទិសដៅ (A <-> B <-> C)
nodeA.next = nodeB
nodeB.prev = nodeA

nodeB.next = nodeC
nodeC.prev = nodeB

# ដើរទៅមុខពី A រហូតដល់ C O(n)
print("១. ដំណើរការដើរទៅមុខ (Forward Traversal):")
current = nodeA
while current:
    print(f"[ {current.data} ]", end=" -> ")
    current = current.next
print("None")

# ដើរថយក្រោយពី C ត្រឡប់មក A វិញ O(n)
print("\n២. ដំណើរការដើរថយក្រោយ (Backward Traversal) តាមរយៈ Prev Pointer:")
current = nodeC
while current:
    print(f"[ {current.data} ]", end=" -> ")
    current = current.prev
print("None")

៨. ការអនុវត្ត Linked Lists ក្នុងប្រព័ន្ធ AI & Data Science
ទោះបីជាភាសា Python ផ្តល់នូវលក្ខណៈងាយស្រួលតាមរយៈ Lists ក៏ដោយ ក៏គំនិតបណ្តាញតភ្ជាប់ (Linked representation) នៅតែជាបេះដូងគ្រឹះនៃស្ថាបត្យកម្មប្រព័ន្ធបញ្ញាសិប្បនិម្មិតកម្រិតខ្ពស់ជាច្រើនដូចខាងក្រោម៖

1. NLP Token Sequences

នៅក្នុងដំណើរការភាសាធម្មជាតិ (NLP) ពាក្យនីមួយៗក្នុងល្បះត្រូវបានបំប្លែងទៅជា Token Nodes។ ម៉ូដែល AI ស្មុគស្មាញត្រូវការការវិភាគបរិបទសងខាង (Symmetric Context) ដោយប្រើប្រាស់គោលការណ៍ Doubly Linked List ដើម្បីឆែកពាក្យនៅខាងមុខ (Next) និងពាក្យពីក្រោយ (Prev) ដើម្បីស្វែងយល់អត្ថន័យ និងវេយ្យាករណ៍បានត្រឹមត្រូវ។

2. Neural Network Layers

រចនាសម្ព័ន្ធបណ្តាញណឺរ៉ូនសិប្បនិម្មិត (ANN) មានស្រទាប់ (Layers) ច្រើនតភ្ជាប់គ្នា។ ការបញ្ជូនទិន្នន័យទៅមុខ (Forward Propagation) និងការគណនាកំហុសថយក្រោយដើម្បីបង្រៀន AI (Backpropagation) ប្រើប្រាស់គោលការណ៍ចង្អុលអាសយដ្ឋានទៅមកដូចជា Doubly Linked List ដើម្បីផ្ទេរទិន្នន័យរវាងស្រទាប់នីមួយៗ។

3. Graph Adjacency Lists

ក្នុងការតំណាងទិន្នន័យក្រាហ្វដែលមាន Nodes រាប់លាន (ដូចជា បណ្តាញទំនាក់ទំនងសង្គម Facebook) គេជៀសវាងការប្រើ Adjacency Matrix (តារាងពីរវិមាត្រ) ព្រោះវាខ្ជះខ្ជាយ Memory O(V²) ខ្លាំង។ ផ្ទុយទៅវិញ គេប្រើ Adjacency List ដែលនៅពីក្រោយខ្នងគឺជាការតម្រៀបនៃ Singly Linked Lists ជួយសន្សំសំចៃ Memory យ៉ាងច្រើន។

4. State Management (Backtracking)

នៅក្នុងការស្វែងរកផ្លូវ និងការដោះស្រាយល្បែងផ្គុំរូបរបស់ AI (ដូចជា ក្បួនដោះស្រាយ DFS ស្វែងរកផ្លូវ) ប្រព័ន្ធត្រូវកត់ត្រាប្រវត្តិទីតាំងនីមួយៗដែលវាបានឆ្លងកាត់។ ការប្រើប្រាស់ Linked List ជួយឲ្យប្រព័ន្ធអាចលុបទីតាំង និងដកថយ (Backtrack) ត្រឡប់ក្រោយបានលឿនកម្រិត O(1) ពេលដែលជួបផ្លូវទាល់។

១. តើភាពខុសគ្នាជាមូលដ្ឋានរវាង Array និង Linked List នៅក្នុង RAM របស់កុំព្យូទ័រមានអ្វីខ្លះ?
• Array: ទាមទារទីតាំងផ្ទុកនៅក្នុង Memory ដែលមានប្លុក **ជាប់ៗគ្នាជានិច្ច (Contiguous Memory)**។ វាដឹងពីទំហំជាក់លាក់ ដែលធ្វើឲ្យការទាញយកលឿន O(1) ប៉ុន្តែមិនបត់បែនពេលពង្រីកទំហំឡើយ។
• Linked List: ផ្ទុកទិន្នន័យនៅទីតាំង **រាយប៉ាយគ្រប់ទិសទី (Scattered Memory)** នៅក្នុង RAM។ វាប្រើប្រាស់ខ្សែចង្អុល (Pointers) ដើម្បីភ្ជាប់ទំនាក់ទំនងពី Node មួយទៅមួយ ធ្វើឲ្យវាមានភាពបត់បែនខ្ពស់ក្នុងការបន្ថែមទិន្នន័យ ប៉ុន្តែការទាញយកមានភាពយឺត O(n)។
២. ហេតុអ្វីបានជាការបញ្ចូលទិន្នន័យនៅខាងមុខបញ្ជី (Head) នៃ Linked List មានល្បឿន O(1) ខណៈពេល Array ត្រូវការ O(n)?
ការបកស្រាយ៖ នៅក្នុង Array ទីតាំងនៃធាតុនីមួយៗត្រូវបានកំណត់ជាប់គ្នានៅក្នុង Memory។ ដូចនេះបើចង់បញ្ចូលទិន្នន័យនៅក្បាល Array កុំព្យូទ័រត្រូវបង្ខំចិត្តរុញធាតុដែលមានស្រាប់ទាំងអស់ទៅក្រោយមួយលំដាប់សិន (ស៊ីពេល O(n))។ ផ្ទុយទៅវិញ នៅក្នុង Linked List យើងគ្រាន់តែបង្កើត Node ថ្មីមួយ រួចកំណត់ឲ្យ Pointer next របស់វាចង្អុលទៅកាន់ Node ក្បាលចាស់ (Head) ហើយដូរ Head មកកាន់ Node ថ្មីនេះវិញជាការស្រេច ដែលប្រើពេលថេរ O(1) ជានិច្ច។
៣. តើលក្ខខណ្ឌណាខ្លះដែលធ្វើឲ្យប្រតិបត្តិការស្វែងរក (Search) ក្នុង Linked List ប្រើពេលយឺត O(n)?

៧. តើអ្វីទៅជា Circular Linked List? តើវាត្រូវបានគេយកទៅប្រើប្រាស់ក្នុងករណីណាខ្លះ?
ការបកស្រាយ៖ Circular Linked List គឺជារចនាសម្ព័ន្ធពិសេសដែល Node ចុងក្រោយគេបង្អស់ (Tail) មិនចង្អុលទៅរក None នោះឡើយ ប៉ុន្តែវាចង្អុលត្រឡប់មកកាន់ Node ទីមួយបង្អស់ (Head) វិញជារង្វង់មូលបិទជិត។ វាត្រូវបានគេប្រើប្រាស់យ៉ាងទូលំទូលាយនៅក្នុងប្រព័ន្ធប្រតិបត្តិការ (OS) សម្រាប់ចាត់ចែងចែកចាយពេលវេលាដំណើរការកម្មវិធីជាច្រើន (Round-Robin CPU Task Scheduling) ឬក្នុងកម្មវិធីហ្គេមដែលត្រូវផ្លាស់ប្តូរវេនគ្នាលេងវិលជុំ។
៨. តើអ្វីទៅជា Garbage Collection នៅក្នុងភាសា Python ហើយវាពាក់ព័ន្ធនឹង Linked List យ៉ាងដូចម្តេច?
ការបកស្រាយ៖ Garbage Collection គឺជាប្រព័ន្ធស្វ័យប្រវត្តិនៃកុំព្យូទ័រក្នុងការសម្អាត និងដកហូតយកទំហំ Memory RAM មកវិញពីវត្ថុ (Objects) ណាដែលលែងមានការប្រើប្រាស់ (Reference Count ស្មើ 0)។ នៅក្នុង Linked List នៅពេលដែលយើងលុប Node មួយចេញដោយផ្តាច់ខ្សែចង Pointer ទាំងអស់ដែលចង្អុលទៅកាន់វា Node នោះនឹងលែងមានអ្នកណាស្គាល់ឡើយ (Reference Count = 0) នោះប្រព័ន្ធ Garbage Collector នឹងស្វែងរក និងបំផ្លាញវាចោលពី RAM ដោយស្វ័យប្រវត្តិ។

ផ្នែកអនុវត្តន៍ជាក់ស្តែងសរុបប្រចាំជំពូកទី ៤៖
បន្ទាប់ពីយល់ដឹងពីទ្រឹស្តី និស្សិតត្រូវសរសេរកូដ Python បង្កើតប្រតិបត្តិការលុប Node (Deletion) និងការបញ្ចូល Node ថ្មីនៅចន្លោះកណ្តាលបញ្ជី។ លំហាត់កូដគំរូ និងការណែនាំពីរបៀបបង្កើតប្រព័ន្ធ History របស់ Web Browser ដោយប្រើប្រាស់ Doubly Linked List ត្រូវបានរៀបចំយ៉ាងលម្អិតនៅក្នុង "សៀវភៅលំហាត់អនុវត្តន៍ (Workbook) - ជំពូកទី ៤"។

១១. សទ្ទានុក្រម និងពាក្យគន្លឹះប្រចាំជំពូកទី ៤ (Key Terms & Glossary)

Node (ថ្នាំង/ប្រអប់ទិន្នន័យ)៖ ជាសមាសភាគវត្ថុ (Object) នីមួយៗនៅក្នុងបណ្តាញ Linked List ដែលផ្ទុកនូវតម្លៃទិន្នន័យ (Data) និងអាសយដ្ឋានចង្អុល (Pointers)។

Head (ក្បាលបញ្ជី)៖ ជា Node ដំបូងគេបង្អស់នៃខ្សែភ្ជាប់ ដែលជាចំណុចចាប់ផ្តើមនៃការស្វែងរក និងដំណើរការទិន្នន័យទាំងអស់ក្នុងបណ្តាញ។

Tail (កន្ទុយបញ្ជី)៖ ជា Node ចុងក្រោយគេបង្អស់ដែលលែងមានការចង្អុលទៅមុខទៀតហើយ គឺចង្អុលទៅរកតម្លៃ None។

Pointer / Reference ៖ ជាអាសយដ្ឋានចង្អុលនៅក្នុង RAM ដែលប្រាប់ប្រព័ន្ធកុំព្យូទ័រថា តើទីតាំងទិន្នន័យរបស់ Node បន្ទាប់ស្ថិតនៅត្រង់ណាពិតប្រាកដ។

None / Null ៖ ជាតម្លៃពិសេសដែលតំណាងឲ្យការ "គ្មានទិន្នន័យ" ឬបញ្ជាក់ពីទីបញ្ចប់នៃខ្សែតភ្ជាប់បណ្តាញ។

Traversing (ការដើររុករក)៖ ជាសកម្មភាពនៃការដើរត្រួតពិនិត្យទិន្នន័យម្តងមួយ Node ពីដើមរហូតដល់ចប់បញ្ជី។

ការយល់ដឹងពីការគ្រប់គ្រង Memory កម្រិត Hardware

ក្នុងនាមជាអ្នកវិទ្យាសាស្ត្រទិន្នន័យ និងវិស្វករ AI ការយល់ច្បាស់ពីយន្តការគ្រប់គ្រង Memory របស់ Linked List ជួយឱ្យយើងចេះថ្លឹងថ្លែងច្បាស់លាស់ពីការប្រើប្រាស់ RAM។ នៅពេលដំណើរការទិន្នន័យខ្នាតយក្ស (Big Data) ការជ្រើសរើសរចនាសម្ព័ន្ធត្រឹមត្រូវមិនត្រឹមតែជួយឲ្យកូដរត់លឿននោះទេ ប៉ុន្តែវាជួយការពារបញ្ហាគាំងប្រព័ន្ធ (Memory Leak/RAM Exhaustion) ថែមទៀតផង។
