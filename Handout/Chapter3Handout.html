១. សេចក្តីផ្តើមអំពីរចនាសម្ព័ន្ធទិន្នន័យជាជួរ (Linear Data Structures)
Linear Data Structures គឺជារចនាសម្ព័ន្ធដែលតម្រៀបទិន្នន័យជាបន្តបន្ទាប់គ្នា (Sequential) ដែលធាតុនីមួយៗត្រូវបានភ្ជាប់ទៅកាន់ធាតុមុន និងធាតុបន្ទាប់របស់វា (លើកលែងតែធាតុដើម និងចុង)។ ទម្រង់នេះមានភាពងាយស្រួលក្នុងការស្វែងរក ពីព្រោះយើងអាចដើរតាមជួរពីដើមដល់ចប់បាន (Traversal)។ នៅក្នុងជំពូកនេះ យើងនឹងសិក្សាអំពីរចនាសម្ព័ន្ធធំៗ ៣ គឺ៖ Arrays, Stacks, និង Queues។
២. រចនាសម្ព័ន្ធ Arrays និងការបែងចែកអង្គចងចាំ
Array គឺជាទម្រង់ទិន្នន័យចំណាស់ជាងគេ និងប្រើប្រាស់ច្រើនជាងគេ។ លក្ខណៈពិសេសរបស់វាគឺ វាទាមទារកន្លែងទំនេរនៅក្នុង Memory កុំព្យូទ័រ ដែលនៅជាប់ៗគ្នា (Contiguous Memory) ដើម្បីផ្ទុកទិន្នន័យ។
ករណីសិក្សា៖ ប្រអប់សំបុត្រ (PO Box)

Array ប្រៀបដូចជាប្រអប់សំបុត្រនៅប្រៃសណីយ៍ដែលតម្រៀបជាប់ៗគ្នា។ កាលណាយើងដឹងពីលេខរៀង (Index) នៃប្រអប់ទី១ យើងអាចលោតទៅចាប់យកប្រអប់ទី១០ បានភ្លាមៗដោយប្រើរូបមន្តគណិតវិទ្យា ធ្វើឲ្យល្បឿនក្នុងការ Access ទិន្នន័យគឺលឿនបំផុតកម្រិត O(1)。

ករណីសិក្សា៖ កៅអីរោងកុន (Static Array)

កៅអីរោងកុនត្រូវបានរៀបចំជាជួរថេរ។ បើជួរនោះមានកៅអី ១០ ហើយមានមនុស្ស ១០ នាក់អង្គុយពេញ បើមានមនុស្សទី ១១ មក គាត់មិនអាចជ្រែកអង្គុយបានទេ លុះត្រាតែយើងប្តូរពួកគេទាំងអស់ទៅជួរថ្មីដែលធំជាងមុន។ នេះហៅថាបញ្ហានៃ Static Array。

បញ្ហានៃការបញ្ចូលទិន្នន័យនៅកណ្តាលជួរ (Insertion Problem)
ការបន្ថែម (Append) ឬលុប (Pop) មនុស្សនៅចុងជួរ គឺលឿនណាស់ (O(1))។ ប៉ុន្តែប្រសិនបើមានអ្នកចង់ជ្រែកចូលនៅកណ្តាលជួរ (កៅអីទី២) យើងត្រូវបង្ខំចិត្តប្រាប់អ្នកអង្គុយពីកៅអីទី២ ដល់ទី១០ ឲ្យងើបឈរ ហើយរំកិលខ្លួនថយក្រោយមួយកៅអីម្នាក់សិន ដើម្បីទុកចន្លោះឲ្យអ្នកថ្មី។ ដំណើរការរុញចុះឡើងនេះ ស៊ីពេលយឺតខ្លាំង O(n)។

៣. ការអនុវត្តកូដ Dynamic Arrays ក្នុង Python
នៅក្នុងភាសា C ឬ Java អ្នកត្រូវប្រាប់កុំព្យូទ័រពីទំហំ Array ជាមុន (ឧ. ផ្ទុកបានតែ ១០ ធាតុ)។ ប៉ុន្តែនៅក្នុងភាសា Python ទិន្នន័យប្រភេទ list គឺជា Dynamic Array។ វាមានប្រព័ន្ធឆ្លាតវៃក្នុងការ "ពង្រីកទំហំខ្លួនឯង (Resize)" ដោយស្វ័យប្រវត្តិនៅពេលដែលវាពេញ ដោយយើងមិនចាំបាច់ខ្វល់ខ្វាយពីការបង្កើត Memory ថ្មីដោយខ្លួនឯងនោះទេ។
1_dynamic_arrays.pyArray Operations
import time

# បង្កើត List (Dynamic Array)
arr = ["Sok", "Sao", "Piseth"]
print(f"១. ទិន្នន័យដើម: {arr}")

# การ Access ទិន្នន័យតាមរយៈ Index គឺលឿនបំផុត O(1)
print(f"២. ហៅឈ្មោះអ្នកទី២: {arr[1]}")

# ការបន្ថែមនៅចុងជួរ (Append) គឺលឿន O(1)
arr.append("Dara")
print(f"៣. ក្រោយបន្ថែមចុង: {arr}")

# ការបញ្ចូលនៅកណ្តាលជួរ (Insert) គឺយឺត O(n)
# ព្រោះ Python ត្រូវរុញ Sao, Piseth, Dara ថយក្រោយ១កន្លែងសិន
arr.insert(1, "Vuthy")
print(f"៤. ក្រោយជ្រែកចូលទី២: {arr}")

# ការវាស់ស្ទង់ភាពយឺតនៃការលុប (Pop) ពីខាងមុខ O(n)
huge_list = list(range(1000000)) # List មាន ១លានតួ

start_time = time.time()
huge_list.pop(0) # លុបអ្នកទី១ចេញ (ត្រូវរុញ ៩សែននាក់ផ្សេងទៀតមកមុខ)
end_time = time.time()

print(f"\n៥. លុបធាតុទី១ ចេញពី List ១លានតួ ប្រើពេល: {end_time - start_time:.5f} វិនាទី")

៣. ក្រោយបន្ថែមចុង: ['Sok', 'Sao', 'Piseth', 'Dara']
៤. ក្រោយជ្រែកចូលទី២: ['Sok', 'Vuthy', 'Sao', 'Piseth', 'Dara']

៥. លុបធាតុទី១ ចេញពី List ១លានតួ ប្រើពេល: 0.00124 វិនាទី

៤. រចនាសម្ព័ន្ធ Stacks (LIFO - Last In, First Out)
Stack ជារចនាសម្ព័ន្ធទិន្នន័យដែលដំណើរការក្រោមច្បាប់ LIFO (អ្នកចូលក្រោយគេ ត្រូវចេញមកមុនគេ)។ ប្រៀបបាននឹងការដាក់របស់ត្រួតលើគ្នា។ យើងអាចបន្ថែម ឬដករបស់ចេញបានតែពីផ្នែកខាងលើ (Top) តែប៉ុណ្ណោះ។

ប្រតិបត្តិការ Push()

ការដាក់ទិន្នន័យថ្មីចូលទៅផ្នែកខាងលើបង្អស់នៃ Stack។ ល្បឿនគឺលឿនបំផុត O(1) ព្រោះមិនចាំបាច់រុញទិន្នន័យចាស់ៗឡើយ។

ប្រតិបត្តិការ Pop()

៥. ការអនុវត្តកូដ Stacks ក្នុង Python
នៅក្នុងភាសា Python យើងមិនចាំបាច់ដំឡើង Library អ្វីថ្មីដើម្បីប្រើ Stack នោះទេ។ យើងអាចយក list ធម្មតាមកប្រើជា Stack បានយ៉ាងល្អឥតខ្ចោះ។ មូលហេតុគឺដោយសារតែការបញ្ចូល append() និងការដកចេញ pop() នៅខាងចុងបញ្ជីរបស់ Python គឺមានល្បឿនលឿនថេរ O(1) ស្រាប់ទៅហើយ។

# បង្កើត Stack ទទេមួយ
history_stack = []

# 1. Push - ដាក់ទិន្នន័យបញ្ជាន់ពីលើគ្នា (O(1))
history_stack.append("ចូលទំព័រ Home")
history_stack.append("ចូលទំព័រ About Us")
history_stack.append("ចូលទំព័រ Contact")

print(f"ស្ថានភាព Stack បច្ចុប្បន្ន: {history_stack}")

# 2. Peek - មើលទិន្នន័យដែលនៅខាងលើគេបង្អស់ (តែមិនដកវាចេញ) (O(1))
# ក្នុង Python យើងប្រើ index -1 ដើម្បីយកតួចុងក្រោយ
top_item = history_stack[-1]
print(f"ទំព័រចុងក្រោយដែលបានចូលគឺ: {top_item}")

# 3. Pop - ចុចប៊ូតុង Back ត្រឡប់ក្រោយ (ដកចេញពីលើគេ) (O(1))
removed_item = history_stack.pop()
print(f"អ្នកបានចុច Back ចាកចេញពីទំព័រ: {removed_item}")
print(f"ស្ថានភាព Stack ក្រោយពី Pop: {history_stack}")

# ការការពារកំហុសពេល Stack ទទេ (Empty Stack)
if len(history_stack) > 0:
    history_stack.pop()
else:
    print("មិនអាចត្រឡប់ក្រោយបានទេ!")

៦. រចនាសម្ព័ន្ធ Queues (FIFO - First In, First Out)
Queue ជារចនាសម្ព័ន្ធទិន្នន័យដែលដំណើរការក្រោមច្បាប់ FIFO (អ្នកមកមុន ត្រូវបម្រើមុន)។ វាតំណាងឲ្យការឈរតម្រង់ជួរ។ ទិន្នន័យថ្មីត្រូវចូលតាម "ខាងកន្ទុយជួរ (Rear)" ហើយទិន្នន័យចាស់ត្រូវចេញតាម "ខាងក្បាលជួរ (Front)"។

ក្នុង Python ការប្រើ list.pop(0) ដើម្បីយកអ្នកខាងមុខចេញ គឺយឺតខ្លាំងកម្រិត O(n) ព្រោះកុំព្យូទ័រត្រូវចំណាយពេលរុញទិន្នន័យដែលនៅសល់ទាំងអស់ទៅមុខម្តងមួយៗ។ បើមានអ្នកតម្រង់ជួរ ១លាននាក់ វានឹងគាំងម៉ាស៊ីន。 ដូច្នេះ វិស្វករតែងតែទាញយក Module ដែលមានល្បឿនលឿនគឺ collections.deque មកប្រើជំនួសវិញ។

from collections import deque

# បង្កើត Queue ទទេមួយ ដោយប្រើ deque (អានថា "ដែក")
print_queue = deque()

# 1. Enqueue - ចូលតម្រង់ជួរខាងកន្ទុយ (O(1))
print_queue.append("ឯកសារ សុខ")
print_queue.append("ឯកសារ សៅ")
print_queue.append("ឯកសារ លីម")

print(f"មានឯកសារកំពុងរង់ចាំ: {print_queue}")

# 2. Dequeue - ព្រីនចេញ និងលុបចេញពីក្បាលជួរ (O(1))
# ការប្រើ popleft() គឺមានប្រសិទ្ធភាព O(1) លឿនជាង pop(0) របស់ list
finished_job = print_queue.popleft()
print(f"-> កំពុងព្រីន: {finished_job}")
print(f"នៅសល់ក្នុងជួរ: {print_queue}")

# បម្រើអ្នកបន្ទាប់ទៀត
print(f"-> កំពុងព្រីន: {print_queue.popleft()}")
print(f"នៅសល់ក្នុងជួរ: {print_queue}")

២. ហេតុអ្វីបានជាការបញ្ចូលទិន្នន័យ (Insert) នៅកណ្តាល Array មានល្បឿនយឺតកម្រិត O(n)?

៣. ហេតុអ្វីយើងមិនគួរប្រើ List ធម្មតា ធ្វើជា Queue នៅក្នុងភាសា Python? តើយើងគួរប្រើអ្វីជំនួស?

៤. នៅក្នុងការស្វែងរកផ្លូវ (Pathfinding) ក្នុងវិស័យ AI តើក្បួន DFS និង BFS ប្រើប្រាស់ Data Structures មួយណាខ្លះ?

៥. តើពាក្យ Push និង Pop ខុសគ្នាពី Enqueue និង Dequeue ដូចម្តេច?
• Push / Pop: ប្រើសម្រាប់ Stack។ Push គឺដាក់បញ្ជាន់ពីលើ, ចំណែក Pop គឺទាញយកអ្នកដែលនៅលើគេចេញមកវិញ។
• Enqueue / Dequeue: ប្រើសម្រាប់ Queue។ Enqueue គឺតម្រង់ជួរនៅខាងកន្ទុយ, ចំណែក Dequeue គឺហៅអ្នកដែលនៅក្បាលជួរមុនគេចេញមកបម្រើ។
៦. តើអ្វីទៅជា Circular Queue? ហើយហេតុអ្វីបានជាគេត្រូវការវា?
ការបកស្រាយ៖ Circular Queue (ជួររង្វង់) គឺជាទម្រង់ពង្រីកនៃ Queue ធម្មតា ដែលធាតុចុងក្រោយត្រូវបានភ្ជាប់ត្រឡប់មកធាតុដើមវិញជារាងរង្វង់។ គេត្រូវការវាដើម្បីដោះស្រាយបញ្ហា "ខាតទំហំ Memory" នៅក្នុង Queue ធម្មតា (ប្រភេទ Static Array) ដែលនៅពេលយើងលុបទិន្នន័យពីខាងមុខ (Dequeue) ចន្លោះទំនេរខាងមុខនោះមិនអាចយកមកប្រើឡើងវិញបានទេ លុះត្រាតែរុញទិន្នន័យទាំងអស់មកមុខ ឬប្រើប្រាស់ Circular Queue នេះដើម្បីអនុញ្ញាតឱ្យធាតុថ្មីចូលមកជំនួសកន្លែងទំនេរខាងមុខវិញ។
៧. តើ Stack ដើរតួនាទីយ៉ាងដូចម្តេចនៅក្នុងដំណើរការដោះស្រាយ Call Stack?
ការបកស្រាយ៖ នៅក្នុងប្រព័ន្ធកុំព្យូទ័រ រាល់ពេលដែលអនុគមន៍ (Function) មួយត្រូវបានហៅ (Call) ព័ត៌មាន និងស្ថានភាពរបស់វាត្រូវបានរក្សាទុកនៅក្នុង "Call Stack" (ដែលជារចនាសម្ព័ន្ធ Stack)។ នៅក្នុង Recursion (អនុគមន៍ហៅខ្លួនឯង) កុំព្យូទ័រនឹងបន្ត Push ស្ថានភាពអនុគមន៍នីមួយៗចូលទៅក្នុង Stack រហូតដល់ជួបលក្ខខណ្ឌបញ្ឈប់ (Base case) ទើបវាចាប់ផ្តើម Pop យកលទ្ធផលត្រឡប់មកវិញម្តងមួយៗតាមលំដាប់ផ្ទុយ (LIFO)។ បើគ្មានលក្ខខណ្ឌបញ្ឈប់ច្បាស់លាស់ទេ វានឹងហៀរ Stack ហៅថា Stack Overflow។

Array: រចនាសម្ព័ន្ធទិន្នន័យដែលផ្ទុកធាតុជាបន្តបន្ទាប់គ្នានៅក្នុង Memory។ Access ទិន្នន័យបានលឿនតាមរយៈ Index (O(1))។

LIFO (Stack): Last In, First Out។ យន្តការអ្នកចូលក្រោយគេ ត្រូវចេញមកបម្រើមុនគេ (ប្រៀបដូចជាការត្រួតចាន)។

FIFO (Queue): First In, First Out។ យន្តការអ្នកចូលមុនគេ ត្រូវចេញមកបម្រើមុនគេ (ប្រៀបដូចជាការតម្រង់ជួរ)។

Deque: អានថា "ដែក" (Double-Ended Queue)។ ជារចនាសម្ព័ន្ធពិសេសក្នុង Python អនុញ្ញាតឲ្យបញ្ចូល ឬដកទិន្នន័យពីចុងសងខាងបានលឿន (O(1))។

DFS / BFS: ក្បួនដោះស្រាយសម្រាប់ស្វែងរកក្នុងក្រាហ្វ ដែល DFS ពឹងផ្អែកលើ Stack ហើយ BFS ពឹងផ្អែកលើ Queue។

ការយល់ដឹងពីគ្រឹះ Hardware

ក្នុងនាមជាវិស្វករ AI កម្រិតខ្ពស់ ការយល់ដឹងពីរបៀបដែលទិន្នន័យត្រូវបានរៀបចំនៅក្នុង RAM (ដូចជា Contiguous Memory របស់ Array ធៀបនឹង Scattered Memory Address) ជួយឱ្យយើងអាចសរសេរកូដដែលមានប្រសិទ្ធភាពខ្ពស់ និងចៀសវាងការខ្ជះខ្ជាយធនធានម៉ាស៊ីន (CPU Cache Misses) ពេលដំណើរការទិន្នន័យខ្នាតយក្ស។
