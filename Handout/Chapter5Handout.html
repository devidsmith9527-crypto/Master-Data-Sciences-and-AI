១. សេចក្តីផ្តើមអំពី Trees និងការរៀបចំទិន្នន័យបែប Hierarchical
ខុសប្លែកពីរចនាសម្ព័ន្ធលីនេអ៊ែរ (Linear Data Structures) ដូចជា Arrays, Stacks, Queues ឬ Linked Lists ដែលតម្រៀបទិន្នន័យជាជួរដេកត្រង់ Tree គឺជារចនាសម្ព័ន្ធទិន្នន័យ **មិនលីនេអ៊ែរ (Non-linear)** ដែលរៀបចំទិន្នន័យក្នុងលក្ខណៈជា **ឋានានុក្រម (Hierarchical/Tree-like structure)**។ វាមាន Nodes តភ្ជាប់គ្នាទៅមកដោយប្រើប្រាស់ទំនាក់ទំនងជា មេ-កូន (Parent-Child)។
ករណីសិក្សាទី១៖ ឋានានុក្រមក្រុមហ៊ុន (Org Chart)

នៅក្នុងក្រុមហ៊ុនមួយ នាយកប្រតិបត្តិ (CEO) គឺជាចំណុចកំពូលគេបង្អស់ (Root)។ CEO ចាត់ចែងបញ្ជាមកកាន់ប្រធាននាយកដ្ឋាននីមួយៗ (Internal Nodes) ហើយប្រធាននាយកដ្ឋានចាត់ចែងការងារបន្តមកកាន់បុគ្គលិកធម្មតា (Leaf Nodes) ដែលគ្មានបុគ្គលិកក្រោមបង្គាប់បន្តឡើយ។

ករណីសិក្សាទី២៖ មែកធាងគ្រួសារ (Family Tree)

ប្រវត្តិគ្រួសារចាប់ផ្តើមដំបូងពីជីដូនជីតា (Root Node) បង្កើតបានជាកូនៗគឺឪពុកម្តាយយើង (Parent Nodes) និងបន្តបង្កើតបានជាជំនាន់ចៅៗ (Child Nodes)។ ទំនាក់ទំនងបែបនេះជួយសម្រួលដល់ការកំណត់អត្តសញ្ញាណ និងប្រភពដើមបានយ៉ាងច្បាស់លាស់។

២. សទ្ទានុក្រម និងលក្ខណៈបច្ចេកទេសរបស់ Tree
ដើម្បីងាយស្រួលយល់ និងសរសេរកូដគ្រប់គ្រង Tree យើងត្រូវចាំពាក្យបច្ចេកទេសសំខាន់ៗទាំងនេះ៖

• Root (ឫស)៖ Node កំពូលគេបង្អស់ដែលគ្មាន Parent ឡើយ។ Tree នីមួយៗមាន Root តែមួយគត់។

• Edge (មែក)៖ ខ្សែតភ្ជាប់រវាង Node ពីរ (តំណាងទំនាក់ទំនង Parent-Child)។

• Leaf (ស្លឹក)៖ Node នៅចុងខាងក្រោមគេបង្អស់ដែលគ្មានកូន (Child Nodes = 0)។

• Parent & Child៖ Node មេ (Parent) ស្ថិតនៅកម្រិតលើ និងចង្អុលទៅកាន់ Node កូន (Child) ដែលស្ថិតនៅកម្រិតក្រោមវា។

• Depth (ជម្រៅ)៖ ចំនួន Edge ពី Root មកកាន់ Node នោះ។

• Height (កម្ពស់)៖ ចំនួន Edge ពី Node នោះទៅកាន់ Leaf ដែលនៅឆ្ងាយបំផុត។

៣. Binary Search Tree (BST) និងច្បាប់តម្រៀបគ្រឹះ
Binary Search Tree (BST) គឺជាប្រភេទពិសេសនៃ Binary Tree (ដើមឈើដែលមានកូនយ៉ាងច្រើនបំផុតពីរ គឺកូនឆ្វេង និងកូនស្តាំ) ដែលទិន្នន័យទាំងអស់ត្រូវបានតម្រៀបរៀបចំឡើងទៅតាមច្បាប់ស្នូលដ៏តឹងរឹងមួយគឺ៖
"រាល់តម្លៃទិន្នន័យនៅក្នុងមែកខាងឆ្វេង (Left Subtree) ត្រូវតែតូចជាង Root ជានិច្ច ហើយរាល់តម្លៃនៅក្នុងមែកខាងស្តាំ (Right Subtree) ត្រូវតែធំជាង Root ជានិច្ច។"
ករណីសិក្សាទី១៖ ការស្វែងរកពាក្យក្នុងវចនានុក្រម

ពេលយើងចង់រកពាក្យចាប់ផ្តើមដោយអក្សរ "M" ហើយវចនានុក្រមបើកចំទំព័រអក្សរ "P"។ ដោយសារយើងដឹងថា M មកមុន P យើងមិនបាច់ខ្ជះខ្ជាយពេលឆែកមើលទំព័រខាងស្តាំ (P ដល់ Z) នោះឡើយ គឺយើងកាត់ចោល ៥០% នៃទំព័រដែលសល់ ហើយតម្រង់ទៅឆ្វេងភ្លាមៗ។

ករណីសិក្សាទី២៖ ការចាត់ថ្នាក់ប្រព័ន្ធស្តុកទំនិញ

ក្នុងឃ្លាំងទំនិញទំនើប ផលិតផលដែលមានលេខកូដសម្គាល់តូចជាង ៥០ ត្រូវបានរក្សាទុកនៅផ្នែកខាងឆ្វេង ហើយលេខកូដធំជាង ៥០ ដាក់នៅផ្នែកខាងស្តាំ។ ប្រព័ន្ធស្វ័យប្រវត្តិនឹងស្វែងរកផលិតផលឃើញភ្លាមៗ O(log n) ដោយមិនបាច់ដើររាប់ពីដើមឃ្លាំងដល់ចុងឃ្លាំងឡើយ។

៤. ការអនុវត្តកូដ BST Insertion ក្នុង Python
ដើម្បីបញ្ចូលទិន្នន័យ (Insert) ទៅក្នុង BST យើងត្រូវប្រៀបធៀបតម្លៃដែលត្រូវបញ្ចូលជាមួយ Node បច្ចុប្បន្ន។ បើតូចជាង ត្រូវរត់ទៅមែកឆ្វេង បើធំជាង ត្រូវរត់ទៅមែកស្តាំ។ ប្រតិបត្តិការនេះធ្វើឡើងបែបដដែលៗ (Recursion) រហូតដល់ជួបកន្លែងទំនេរ (None) ទើបបង្កើត Node ថ្មីនៅទីនោះ។
1_bst_insertion.pyBST Node and Insertion Code
class TreeNode:
    def __init__(self, key):
        self.left = None   # កូនឆ្វេង
        self.right = None  # កូនស្តាំ
        self.val = key     # តម្លៃទិន្នន័យ

# អនុគមន៍បញ្ចូលទិន្នន័យ O(log n)
def insert(root, key):
    # ១. បើជួបកន្លែងទំនេរ បង្កើត Node ថ្មីត្រង់នោះ
    if root is None:
        return TreeNode(key)

    # ២. បើតម្លៃតូចជាង Root ត្រូវតម្រង់ទៅឆ្វេង
    if key < root.val:
        root.left = insert(root.left, key)
    # ៣. បើតម្លៃធំជាង Root ត្រូវតម្រង់ទៅស្តាំ
    elif key > root.val:
        root.right = insert(root.right, key)

    return root

# សាងសង់ Tree គំរូ
root = TreeNode(50)
insert(root, 30)
insert(root, 70)
insert(root, 20)

print("--- បញ្ចូលទិន្នន័យ [50, 30, 70, 20] ចូលក្នុង BST បានជោគជ័យ ---")
print(f"តម្លៃ Root កំពូល: {root.val}")
print(f"តម្លៃមែកខាងឆ្វេងរបស់ Root: {root.left.val}")
print(f"តម្លៃមែកខាងស្តាំរបស់ Root: {root.right.val}")

៦. ការអនុវត្តកូដ BST Search ក្នុង Python
ខាងក្រោមនេះជាកូដ Python ក្នុងការបង្កើតអនុគមន៍ស្វែងរកទិន្នន័យ (Search Function) តាមបែបដដែលៗ (Recursion)។ អនុគមន៍នេះនឹងបន្តស្វែងរកដោយច្រោះចោលមែកដែលមិនពាក់ព័ន្ធ ៥០% ចេញរាល់ពេលប្រៀបធៀបម្តងៗ។
2_bst_search.pyBST Search Function Implementation
class TreeNode:
    def __init__(self, key):
        self.left = None; self.right = None
        self.val = key

def insert(root, key):
    if root is None: return TreeNode(key)
    if key < root.val: root.left = insert(root.left, key)
    elif key > root.val: root.right = insert(root.right, key)
    return root

# អនុគមន៍ស្វែងរកទិន្នន័យក្នុង BST O(log n)
def search(root, key):
    # លក្ខខណ្ឌបញ្ចប់ (Base Cases): រកមិនឃើញ (None) ឬ រកឃើញតម្លៃស្មើពិតប្រាកដ
    if root is None or root.val == key:
        return root

    # បើតម្លៃដែលរកធំជាង Root ត្រូវទៅរកនៅមែកស្តាំ
    if key > root.val:
        return search(root.right, key)

    # បើតម្លៃដែលរកតូចជាង Root ត្រូវទៅរកនៅមែកឆ្វេង
    return search(root.left, key)

# រៀបចំទិន្នន័យ
tree = None
for val in [50, 30, 70, 20, 40]:
    tree = insert(tree, val)

# សាកល្បងស្វែងរកលេខ ៤០ (មានក្នុង Tree)
result1 = search(tree, 40)
print(f"ស្វែងរកលេខ 40: {'រកឃើញ!' if result1 else 'រកមិនឃើញទេ!'}")

# សាកល្បងស្វែងរកលេខ ៩៩ (គ្មានក្នុង Tree)
result2 = search(tree, 99)
print(f"ស្វែងរកលេខ 99: {'រកឃើញ!' if result2 else 'រកមិនឃើញទេ!'}")
>_ Terminal Output
ស្វែងរកលេខ 40: រកឃើញ!
ស្វែងរកលេខ 99: រកមិនឃើញទេ!

៧. គោលការណ៍រុករកបង្ហាញទិន្នន័យ (Tree Traversals)

១. In-order Traversal (ឆ្វេង -> ឫស -> ស្តាំ)

រុករកមែកខាងឆ្វេងឲ្យអស់ជាមុន ទើបបង្ហាញ Root រួចទើបបន្តទៅមែកខាងស្តាំ។
• ករណីសិក្សា៖ សម្រាប់ការអានសៀវភៅតាមលំដាប់ទំព័រ ឬការតម្រៀបសិស្សតាមលំដាប់កម្ពស់។ នៅក្នុង BST វិធីសាស្ត្រ In-order នឹងទាញយកទិន្នន័យចេញមកតម្រៀបពី **តូចទៅធំ** បានយ៉ាងស្អាតឥតខ្ចោះជានិច្ច។

២. Pre-order Traversal (ឫស -> ឆ្វេង -> ស្តាំ)

បង្ហាញតម្លៃ Root ជាមុនសិន រួចទើបបន្តទៅរុករកមែកឆ្វេង និងមែកស្តាំតាមលំដាប់។
• ករណីសិក្សា៖ ប្រើប្រាស់សម្រាប់ដំណើរការចម្លងរចនាសម្ព័ន្ធ (Clone/Duplicate Tree) ព្រោះវាត្រូវបង្កើត Node មេជាមុនសិន ទើបអាចបង្កើត Node កូនៗជាបន្តបន្ទាប់នៅពីក្រោមបាន។

៣. Post-order Traversal (ឆ្វេង -> ស្តាំ -> ឫស)

រុករកមែកឆ្វេង និងមែកស្តាំឲ្យអស់ជាមុនសិន រួចទើបបង្ហាញតម្លៃ Root នៅចុងក្រោយគេបង្អស់។
• ករណីសិក្សា៖ ប្រើប្រាស់សម្រាប់ការលុប Folder ចេញពីកុំព្យូទ័រ (Directory Deletion)។ យើងមិនអាចលុប Folder មេបានទេ ប្រសិនបើមិនទាន់បានលុបឯកសារកូនៗ (Files) នៅខាងក្នុងឲ្យអស់ជាមុននោះ (លុបពីក្រោមឡើងលើ)។

3_tree_traversals.pyIn-order, Pre-order and Post-order
class Node:
    def __init__(self, key):
        self.left = None; self.right = None
        self.val = key

# ១. In-order Traversal (ឆ្វេង -> ឫស -> ស្តាំ)
def print_inorder(root):
    if root:
        print_inorder(root.left)
        print(root.val, end=" ")
        print_inorder(root.right)

# ២. Pre-order Traversal (ឫស -> ឆ្វេង -> ស្តាំ)
def print_preorder(root):
    if root:
        print(root.val, end=" ")
        print_preorder(root.left)
        print_preorder(root.right)

# ៣. Post-order Traversal (ឆ្វេង -> ស្តាំ -> ឫស)
def print_postorder(root):
    if root:
        print_postorder(root.left)
        print_postorder(root.right)
        print(root.val, end=" ")

# បង្កើត BST គំរូ
root = Node(50)
root.left = Node(30); root.right = Node(70)
root.left.left = Node(20); root.left.right = Node(40)

print("In-order (តម្រៀបលេខពីតូចទៅធំ):")
print_inorder(root)
print("\n\nPre-order (ល្អសម្រាប់ចម្លង Tree):")
print_preorder(root)
print("\n\nPost-order (ល្អសម្រាប់លុប Folder):")
print_postorder(root)

រចនាសម្ព័ន្ធទិន្នន័យ Trees មិនត្រឹមតែជាទ្រឹស្តីកូដនោះទេ វាគឺជាគ្រឹះស្នូលនៃស្ថាបត្យកម្មប្រព័ន្ធកុំព្យូទ័រ និងបញ្ញាសិប្បនិម្មិតកម្រិតខ្ពស់ជាច្រើន៖

1. Database Indexing (MySQL/NoSQL)

នៅក្នុងប្រព័ន្ធគ្រប់គ្រងទិន្នន័យខ្នាតយក្ស ដូចជា MySQL គេប្រើប្រាស់រចនាសម្ព័ន្ធ B-Trees & B+ Trees (ដែលជាទម្រង់ពង្រីកខ្ពស់នៃ Binary Search Tree) ដើម្បីធ្វើសន្ទស្សន៍ (Indexing)។ វាជួយឲ្យប្រព័ន្ធស្វែងរកទិន្នន័យរាប់លានជួរ (Records) បានលឿនក្នុងល្បឿន O(log n) ដោយមិនបាច់ដើរ Scan គ្រប់ទិន្នន័យទាំងអស់ក្នុង Disk ឡើយ។

2. Decision Tree Models (ML)

នៅក្នុង Machine Learning, Decision Tree (ដើមឈើសម្រេចចិត្ត) គឺជាក្បួនដោះស្រាយដ៏ល្បីល្បាញសម្រាប់ធ្វើចំណាត់ថ្នាក់ទិន្នន័យ (Classification) និងការទស្សន៍ទាយលទ្ធផល (Regression)។ AI នឹងបង្កើតលក្ខខណ្ឌវិនិច្ឆ័យនៅតាម Nodes នីមួយៗ (ឧ. "តើប្រាក់ចំណូល > $៥០០ ឬទេ?") រហូតដល់រកឃើញចម្លើយសម្រេចចិត្តនៅចុងបញ្ចប់ (Leaf Nodes)។

3. Game AI (Minimax Algorithm)

នៅក្នុងការបង្កើត AI លេងហ្គេមឆ្លាតវៃដូចជា ហ្គេមអុក (Chess) ឬ Tic-Tac-Toe គេប្រើប្រាស់ Game Tree & Minimax Algorithm។ AI នឹងបង្កើតទម្រង់ដើមឈើគណនាទឹកដើររបស់គូប្រកួត (Root តំណាងទឹកដើរបច្ចុប្បន្ន ឯកូនៗតំណាងជម្រើសទឹកដើរដែលអាចទៅរួច) រួចធ្វើការជ្រើសរើសទឹកដើរណាដែលអាចឈ្នះ ឬខាតបង់តិចបំផុត។

4. K-D Trees for Spatial Queries

នៅក្នុងប្រព័ន្ធផែនទីភូមិសាស្ត្រ (GIS) និងការស្រាវជ្រាវ AI លើបណ្តាញផែនទី (ដូចជា Google Maps ឬប្រព័ន្ធបើកបរស្វ័យប្រវត្តិនៃឡាន Tesla) គេប្រើប្រាស់ K-Dimensional Trees (K-D Trees) ដើម្បីស្វែងរកចំណុចដែលនៅជិតបំផុត (Nearest Neighbor Search) ជួយឲ្យរ៉ូបូតដឹងពីវត្ថុរារាំងនៅជិតខ្លួនបានលឿនបំផុត O(log n)។

១. តើរចនាសម្ព័ន្ធ Tree ខុសប្លែកពី Linked List យ៉ាងដូចម្តេចពិតប្រាកដ?
• Linked List: ជារចនាសម្ព័ន្ធបែបលីនេអ៊ែរ (Linear) ដែល Node នីមួយៗភ្ជាប់ទៅកាន់ Node បន្ទាប់ត្រង់តែមួយគត់ (ទំនាក់ទំនងបែប ១ ទៅ ១)។
• Tree: ជារចនាសម្ព័ន្ធមិនលីនេអ៊ែរ (Non-linear) បែបឋានានុក្រម ដែល Node មេ (Parent) មួយអាចចង្អុលភ្ជាប់ទៅកាន់ Node កូនៗជាច្រើននៅពីក្រោម (ទំនាក់ទំនងបែប ១ ទៅច្រើន)។
២. តើច្បាប់ស្នូលក្នុងការរៀបចំទិន្នន័យរបស់ Binary Search Tree (BST) មានអត្ថន័យដូចម្តេច?
ការបកស្រាយ៖ នៅក្នុង BST គ្រប់ Node ទាំងអស់ត្រូវគោរពតាមច្បាប់៖ តម្លៃទិន្នន័យនៅមែកខាងឆ្វេង (Left Child) របស់វាទាំងអស់ ត្រូវតែមានតម្លៃ **តូចជាង** ខ្លួនវាជានិច្ច ហើយតម្លៃទិន្នន័យនៅមែកខាងស្តាំ (Right Child) ទាំងអស់ ត្រូវតែមានតម្លៃ **ធំជាង** ខ្លួនវាជានិច្ច។ ច្បាប់នេះជួយឱ្យកុំព្យូទ័រអាចស្វែងរក និងបញ្ចូលទិន្នន័យបានលឿនបំផុតកម្រិត O(log n)។
៣. ហេតុអ្វីបានជាការរុករកបែប In-order Traversal លើ BST តែងតែបោះពុម្ពតម្លៃចេញមកតម្រៀបពីតូចទៅធំ?
ការបកស្រាយ៖ ដោយសារយន្តការ In-order ដំណើរការទៅតាមលំដាប់៖ **(មែកឆ្វេង -> Root -> មែកស្តាំ)**។ យោងតាមច្បាប់ BST តម្លៃតូចបំផុតស្ថិតនៅខាងឆ្វេងបំផុត តម្លៃកណ្តាលគឺ Root ហើយតម្លៃធំបំផុតស្ថិតនៅខាងស្តាំបំផុត។ ដូចនេះនៅពេលអនុវត្តតាម In-order កុំព្យូទ័រនឹងដំណើរការឆ្វេង (តូច) រួចមកកណ្តាល រួចទើបទៅស្តាំ (ធំ) ដែលធ្វើឱ្យលទ្ធផលបោះពុម្ពចេញមកតម្រៀបគ្នាពីតូចទៅធំជានិច្ច។
៤. តើអ្វីទៅជា Skewed Tree នៅក្នុង BST? ហើយតើវាជះឥទ្ធិពលអាក្រក់អ្វីខ្លះដល់ល្បឿនដំណើរការប្រព័ន្ធ?
ការបកស្រាយ៖ Skewed Tree គឺជា Tree ដែលដុះមែកតម្រង់ទៅម្ខាងគត់ (ដុះទៅឆ្វេងតែម្ខាង ឬទៅស្តាំតែម្ខាង) ដែលកើតឡើងនៅពេលយើងបញ្ចូលទិន្នន័យដែលតម្រៀបរួចជាស្រេច (Sorted Data) ដូចជា [10, 20, 30, 40]។ ឥទ្ធិពលអាក្រក់គឺវាធ្វើឱ្យកម្ពស់ Tree កើនឡើងស្មើនឹង $n$ ធាតុ បណ្តាលឱ្យល្បឿនស្វែងរកធ្លាក់ចុះមកកម្រិតអាក្រក់បំផុតគឺ O(n) (ស្មើ Linked List ធម្មតា) គឺបាត់បង់ប្រសិទ្ធភាព O(log n) ទាំងស្រុង។

៥. តើ In-order, Pre-order, និង Post-order ខុសគ្នាយ៉ាងដូចម្តេចក្នុងការសរសេរកូដ Recursion?
ការបកស្រាយ៖ ភាពខុសគ្នាគឺស្ថិតនៅលើ **ទីតាំងនៃការបោះពុម្ពតម្លៃ Root** បើធៀបនឹងការហៅអនុគមន៍ដំណើរការឆ្វេង និងស្តាំ៖
• In-order: ហៅមែកឆ្វេង រួច **បោះពុម្ព Root (កណ្តាល)** រួចហៅមែកស្តាំ។
• Pre-order: **បោះពុម្ព Root (ដំបូងគេ)** រួចហៅមែកឆ្វេង រួចហៅមែកស្តាំ។
• Post-order: ហៅមែកឆ្វេង រួចហៅមែកស្តាំ រួចទើប **បោះពុម្ព Root (ចុងក្រោយគេ)**។
៦. ហេតុអ្វីបានជា Database ល្បីៗដូចជា MySQL ប្រើប្រាស់រចនាសម្ព័ន្ធ B-Tree ជំនួសឱ្យ BST ធម្មតា?
ការបកស្រាយ៖ BST ធម្មតាអនុញ្ញាតឲ្យ Node នីមួយៗមានកូនយ៉ាងច្រើនបំផុតតែ ២ គត់ (Binary) ដែលធ្វើឱ្យកម្ពស់ Tree កើនឡើងខ្ពស់ខ្លាំងពេលទិន្នន័យឡើងដល់រាប់លាន នាំឱ្យចំណាយពេលចុះអានទិន្នន័យពី Memory ច្រើនដង។ ផ្ទុយទៅវិញ B-Tree អនុញ្ញាតឲ្យ Node នីមួយៗផ្ទុកទិន្នន័យបានរាប់រយតម្លៃ និងមានកូនរាប់រយមែកធាងក្នុងកម្រិតតែមួយ ជួយកាត់បន្ថយកម្ពស់ Tree មកទាបបំផុត (Flat Tree) ដែលសាកសមបំផុតសម្រាប់ការស្វែងរកទិន្នន័យខ្នាតយក្សក្នុង Hard Disk O(log n)។
១២. សទ្ទានុក្រម និងពាក្យគន្លឹះប្រចាំជំពូកទី ៥ (Glossary)

• Tree (ដើមឈើ)៖ រចនាសម្ព័ន្ធទិន្នន័យមិនលីនេអ៊ែរបែបឋានានុក្រម តំណាងដោយការតភ្ជាប់គ្នានៃ Nodes។

• Root Node (ឫស)៖ Node កំពូលនៃ Tree ដែលជាចំណុចចាប់ផ្តើមនៃការស្វែងរកទិន្នន័យទាំងអស់។

• Internal Node៖ Node ដែលស្ថិតនៅចន្លោះកណ្តាល (មានទាំង Parent និង Children)។

• Leaf Node (ស្លឹក)៖ Node នៅចុងខាងក្រោមគេបង្អស់ ដែលគ្មានកូនបន្តទៀតឡើយ (Children = 0)។

• BST (Binary Search Tree)៖ Binary Tree ដែលរក្សាច្បាប់៖ មែកឆ្វេងតូចជាង Root មែកស្តាំធំជាង Root ជានិច្ច។

• Skewed Tree៖ ស្ថានភាព Tree អតុល្យភាព ដែលដុះទៅខាងឆ្វេង ឬខាងស្តាំតែម្ខាង នាំឱ្យល្បឿនធ្លាក់ចុះមក O(n)។

• Traversal (ការរុករក)៖ យន្តការដើរឆែកមើលទិន្នន័យគ្រប់ Nodes ទាំងអស់ក្នុង Tree (In-order, Pre-order, Post-order)។
