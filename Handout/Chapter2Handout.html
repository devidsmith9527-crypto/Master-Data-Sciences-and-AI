២. និយមន័យ Class និង Object

Class (គំរូ / ប្លង់)

គឺជាកូដដែលដើរតួជា "ប្លង់ (Blueprint)" សម្រាប់កំណត់ពីលក្ខណៈសម្បត្តិ (Properties/Attributes) និងមុខងារ (Methods) សម្រាប់បង្កើតវត្ថុអ្វីមួយ ប៉ុន្តែវាមិនមានរូបរាង ឬមិនទាន់ស៊ីទំហំក្នុងអង្គចងចាំ (Memory) នោះទេ។

Object (វត្ថុជាក់ស្តែង)

គឺជាវត្ថុ (Instance) ដែលកើតចេញពីការសាងសង់ដោយផ្អែកលើប្លង់ (Class) នោះ។ វាមានរូបរាង មានទិន្នន័យពិតប្រាកដ និងស៊ីទំហំនៅក្នុង Memory (Heap) របស់កុំព្យូទ័រ។

ឧទាហរណ៍៖ ប្លង់ផ្ទះ និងការសាងសង់ផ្ទះ

វិស្វករគូរ "ប្លង់ផ្ទះ" (Class) នៅលើក្រដាស ដោយកំណត់ថាផ្ទះត្រូវមានទ្វារ និងបង្អួច ប៉ុន្តែអ្នកមិនអាចចូលទៅដេកក្នុងក្រដាសប្លង់នោះបានទេ។ នៅពេលអ្នកម៉ៅការយកប្លង់នោះទៅ "សាងសង់ជាផ្ទះទី១ ផ្ទះទី២" នោះទើបក្លាយជា "វត្ថុពិត (Objects)" ដែលអាចរស់នៅបាន។ ប្លង់១ អាចបង្កើតផ្ទះបានរាប់ពាន់ខ្នង។

៣. ការអនុវត្តបង្កើត Class សម្រាប់ Node (ចំណុចផ្ទុក)
ដើម្បីរៀបចំទិន្នន័យជាបណ្តាញភ្ជាប់គ្នា (Linked List) យើងត្រូវការ `Class Node` ដែលមានតួនាទី ២ យ៉ាងគឺ ផ្ទុកទិន្នន័យ (Data) និង ចង្អុលទៅអ្នកបន្ទាប់ (Next pointer)។
1_class_node.pyOOP Basic
class Node:
    # Constructor (អនុគមន៍សាងសង់ Object ពេលហៅឈ្មោះ Class)
    def __init__(self, value):
        self.data = value   # កន្លែងផ្ទុកទិន្នន័យ
        self.next = None    # ខ្សែចង្អុលទៅប្រអប់បន្ទាប់ (លំនាំដើមគឺ None)

# បង្កើត Objects ពិតប្រាកដចេញពី Class Node
node1 = Node("Dara")
node2 = Node("Sok")

# ភ្ជាប់ Object ទី១ ទៅ Object ទី២
node1.next = node2

print(f"ប្រអប់ទី១ មានទិន្នន័យ: {node1.data}")
print(f"ប្រអប់ទី១ ចង្អុលទៅទិន្នន័យ: {node1.next.data}")
>_ Terminal Output
ប្រអប់ទី១ មានទិន្នន័យ: Dara
ប្រអប់ទី១ ចង្អុលទៅទិន្នន័យ: Sok

៤. សសរទ្រូងទាំង ៤ នៃ Object-Oriented Programming

១. ការវិចខ្ចប់ និងលាក់បាំងទិន្នន័យ (Encapsulation)
ទ្រឹស្តី និងជីវិតពិត
ជាការលាក់បាំងទិន្នន័យសំខាន់ៗ មិនឲ្យកូដពីខាងក្រៅចូលមកកែប្រែដោយផ្ទាល់បាន។ ឧទាហរណ៍៖ ទូដែក (Safe Box) លាក់លុយនៅខាងក្នុង។ អ្នកមិនអាចលូកដៃចាប់យកលុយបានទេ លុះត្រាតែអ្នកវាយលេខកូដ (Method) ឲ្យបានត្រឹមត្រូវទើបទូដែកបើកឲ្យ។
ការប្រើប្រាស់ក្នុង Python

១១. សទ្ទានុក្រម និងពាក្យគន្លឹះ (Key Terms & Glossary)

Class / Object: Class គឺប្លង់គំរូនៃកូដ ខណៈ Object (Instance) គឺជាវត្ថុពិតប្រាកដដែលកើតចេញពីប្លង់នោះ និងមានទិន្នន័យជាក់ស្តែង។

Inheritance: យន្តការដែលអនុញ្ញាតឲ្យ Class ថ្មីទទួលបានលក្ខណៈសម្បត្តិពី Class ចាស់ ដោយមិនចាំបាច់សរសេរកូដសារឡើងវិញ។

Pointer / Reference: ឈ្មោះអញ្ញាតដែលធ្វើការចង្អុលទីតាំងទៅរក Object នៅក្នុង Memory (Heap)។

Garbage Collection (GC): ដំណើរការដែលប្រព័ន្ធកុំព្យូទ័រតាមដាន និងលុបចោល Object ណាដែលគ្មាន Variable ណាបញ្ជូនទៅកាន់វាទៀត។

Shallow Copy: ការថតចម្លងតែកម្រិតស្រទាប់ក្រៅ ដែលស្រទាប់ខាងក្នុងៗនៅតែចែករំលែក Memory ដដែល។

Deep Copy: ការក្លូន Object និងធាតុទាំងអស់ខាងក្នុង បង្កើតជាឯកសារថ្មីទាំងស្រុងដែលមិនពាក់ព័ន្ធគ្នា។

១២. សេចក្តីសន្និដ្ឋាននៃជំពូកទី ២ (Conclusion)
  • OOP គឺជាគ្រឹះនៃ Data Structures ផ្ទាល់ខ្លួន: ដើម្បីបង្កើត Linked List, Tree ឬ Graph ដែលជាទម្រង់ទិន្នន័យសំខាន់បំផុតក្នុង AI យើងមិនអាចខ្វះការយល់ដឹងពីការសរសេរ Class នោះទេ។ វាផ្តល់សេរីភាពឲ្យយើងរៀបចំកូដបានតាមតម្រូវការ។
  • ការយល់ដឹងពី Memory ទប់ស្កាត់កំហុសធ្ងន់ធ្ងរ: ការយល់ថា Variables ក្នុង Python គ្រាន់តែជា "អ្នកចង្អុល (References)" ជួយឲ្យអ្នកវិភាគទិន្នន័យជៀសវាងការធ្វើឲ្យខូចខាត (Mutate) ទៅលើ Dataset ដើមពេលបញ្ជូនទិន្នន័យចូលទៅក្នុង Functions ផ្សេងៗ។
  • ផ្តាច់ខ្លួនដោយ Deep Copy: នៅក្នុងគម្រោងធំៗ ការចម្លងទិន្នន័យដោយប្រើសញ្ញាស្មើ = គឺជាហានិភ័យខ្ពស់។ ចូរចងចាំថា ការការពារទិន្នន័យ 2D ឬ 3D (ដូចជារូបភាព ឬម៉ាទ្រីស) ទាមទារឲ្យមានការប្រើប្រាស់ Deep Copy ជានិច្ច ដើម្បីធានាថាការពិសោធន៍ថ្មីៗមិនប៉ះពាល់ដល់ទិន្នន័យស្តង់ដារ។
